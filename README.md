# MaiBot AI 对话查询插件

MaiBot 插件，提供自助 AI 对话状态查询、申请开启、管理员私聊审批功能。通过 NapCat 第二 WebSocket 端口直连，与主适配器互不干扰。

## 特性

- **查询状态** — 发送命令查看群/私聊是否已开启 AI 对话
- **自助申请** — 用户发送命令提交开通申请，通知管理员审批
- **私聊审批** — 管理员在私聊中通过/拒绝（支持附带拒绝理由），自动写入 NapCat 白名单
- **群聊通知** — 审批结果同步通知到对应群聊
- **命令全自定义** — 所有触发命令均可自定义，支持开启/关闭 `/` 前缀
- **昵称驱动** — 回复文案中的机器人名称读取 `bot_nickname` 配置
- **安全写入** — 修改 NapCat 配置文件前自动备份，写入后校验 TOML 合法性，失败自动回滚
- **持久化** — 申请队列保存为 JSON，重启不丢失

## 安装

将整个 `ai-chat-query/` 目录放入 MaiBot 的 `plugins/` 目录下：

```
plugins/
├── ai-chat-query/
│   ├── plugin.py
│   ├── config.toml
│   ├── config_models.py
│   ├── command_parser.py
│   ├── ws_client.py
│   ├── queue.py
│   ├── _manifest.json
│   └── README.md
└── maibot-team_napcat-adapter/
    └── config.toml
```

## 配置 NapCat 第二端口

在 NapCat 的 `onebot11_<QQ号>.json` 中添加第二个 WebSocket 服务端：

```json
{
    "name": "AI查询插件端口",
    "port": 3002,
    "enable": true,
    "heartbeatInterval": 30000,
    "token": ""
}
```

重启 NapCat 后生效。端口号需与插件 `config.toml` 中 `ws_port` 一致。

## 配置说明

编辑 `plugins/ai-chat-query/config.toml`：

### `[plugin]`

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `enabled` | `true` | 是否启用插件 |

### `[napcat]`

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `ws_host` | `127.0.0.1` | NapCat 第二端口的地址 |
| `ws_port` | `3002` | NapCat 第二端口的端口号 |
| `token` | `""` | NapCat 鉴权 Token，无鉴权则留空 |

### `[commands]`

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `query_commands` | `["/查询ai聊天"]` | 查询命令，可多个 |
| `apply_commands` | `["/开启ai聊天"]` | 申请命令，可多个 |
| `approve_commands` | `["/通过申请"]` | 管理员通过命令 |
| `reject_commands` | `["/拒绝申请"]` | 管理员拒绝命令 |
| `list_all_commands` | `["/查申请"]` | 查看待审批列表命令 |
| `enable_slash_prefix` | `true` | 命令是否以 `/` 开头 |

> 设置自定义命令后，默认命令失效。例如 `query_commands = ["查状态", "状态"]` 后仅这两个词可触发查询。

### `[admin]`

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `super_admin_qq` | `""` | 管理员 QQ 号，审批仅限此账号私聊 |
| `napcat_config_path` | `""` | NapCat 适配器配置文件路径，留空自动检测 |

### `[reply]`

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `bot_nickname` | `""` | 机器人昵称，用于回复文案，留空则自动获取 |

## 命令

### 普通用户

| 命令（默认） | 适用 | 说明 |
|-------------|------|------|
| `/查询ai聊天` | 群聊/私聊 | 查看是否已开启 |
| `/开启ai聊天` | 群聊/私聊 | 提交开通申请 |

### 管理员（仅私聊）

| 命令（默认） | 说明 |
|-------------|------|
| `/通过申请` | 通过最早一条待审批 |
| `/通过申请 <编号>` | 通过指定编号 |
| `/拒绝申请` | 拒绝最早一条待审批 |
| `/拒绝申请 <编号> <理由>` | 拒绝并附理由 |
| `/查申请` | 查看所有待审批 |

## 工作流程

```
用户发送 /开启ai聊天
  → 检查是否已在白名单 → 是：提示"已开启"
  → 检查是否有待审批   → 是：提示"等待审核"
  → 创建申请，通知管理员私聊

管理员私聊 /通过申请 3
  → 验证管理员身份
  → 备份 NapCat 配置文件
  → 追加群号/QQ 到白名单
  → 校验 TOML → 写入 → 失败则回滚
  → 通知管理员 + 通知申请人 + 通知群聊
```
